# 公共约定（所有模块通用）

> 各模块接口文档仅描述本模块特有的数据模型与接口，**通用约定统一以本文档为准**。

---

## 1. 接口基础地址

| 环境 | 基础地址 | 说明 |
|---|---|---|
| **开发（走网关）** | `http://localhost:37050` | ✅ 前端联调推荐，与生产链路一致 |
| 开发（直连服务） | `http://localhost:37056` | 仅联调期绕过网关鉴权时使用 |

业务路径前缀：`/api/{模块}`

## 2. 统一响应格式

```json
{
  "code": 0,
  "data": { ... },
  "message": null
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | int | `0` = 成功，非 `0` = 失败 |
| `data` | any | 业务数据（对象 / 数组 / 分页对象 / `null`） |
| `message` | string \| null | 成功通常 `null`；失败为错误描述 |

> ✅ **判断准则**：`code === 0` 成功；`code !== 0` 时读 `message`。

## 3. 分页约定

**请求参数**（Query）：

| 参数 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| `page` | int | 否 | `1` | 页码，从 1 开始 |
| `pageSize` | int | 否 | `20` | 每页条数，上限 500 |
| `sortBy` | string | 否 | 模块默认 | 排序字段，见各模块 |
| `order` | string | 否 | 模块默认 | `asc` / `desc` |

**响应 `data`**（`PageResult`）：

```json
{ "list": [...], "total": 100, "page": 1, "pageSize": 20 }
```

## 4. 鉴权

除白名单接口外，请求 Header 需携带：
```
Authorization: Bearer <token>
```
联调被 401 阻塞时，可临时用直连地址（37056）绕过网关。

## 5. 数据类型

| 类型 | JSON 表现 | 示例 |
|---|---|---|
| `Long`（主键 id） | 数字 | `1` |
| `Integer`（排序/计数） | 数字 | `10` |
| `Boolean`（启用状态） | 布尔 | `true` / `false` |
| `LocalDateTime` | 字符串带 `T` | `"2026-07-01T14:18:11"` |
| `String` | 字符串 | `"https://..."` |

## 6. RESTful 路径约定

| 操作 | 方法 & 路径 | 说明 |
|---|---|---|
| 列表 | `GET /api/{模块}` | Query 参数 |
| 新增 | `POST /api/{模块}` | body 不含 id，**返回含 id 的对象** |
| 修改 | `PUT /api/{模块}/{id}` | id 在 path，body 不含 id |
| 删除 | `DELETE /api/{模块}/{id}` | 逻辑删除（`is_deleted=1`） |

## 7. 错误码

| code | 含义 | message 示例 | 前端处理 |
|---|---|---|---|
| `0` | 成功 | `null` | 处理 `data` |
| `-1` | 业务失败 | `"修改失败"` | 弹出 `message` |
| `-1` | 参数校验失败 | `"名称不能为空"` | 表单内联提示或弹出 |
| `401` | 未认证 | - | 跳转登录 |

异常响应同结构：`{ "code": -1, "data": null, "message": "..." }`

## 8. 通用联调注意

1. 响应成功码是 `0`，不是 `200`；字段是 `message`，不是 `msg`。
2. 启用状态用布尔 `enabled`（`true`/`false`），不是 0/1，不叫 status。
3. 修改/删除的 id 在 path，不在 body。
4. 新增（POST）成功返回含 id 的对象。
5. 时间格式带 `T`，用 day.js 格式化展示。
6. 删除是逻辑删除：列表查不到，DB 保留，误删联系后端恢复。
7. `id` 是数字类型，JS 按 Number 处理。
